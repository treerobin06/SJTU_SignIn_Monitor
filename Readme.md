# 上海交通大学Canvas签到号监控脚本

## 📖 项目简介

这是一个基于Python Selenium的自动化监控脚本，用于监控上海交通大学Canvas系统中的签到号变化。当检测到签到号发生变化时，脚本会发出声音警报提醒用户。

## 🏗️ 代码结构说明

### 主要模块功能

| 模块/方法 | 功能说明 |
|-----------|----------|
| `SigninMonitorV3` | 主监控类，包含所有监控逻辑 |
| `setup_driver()` | 初始化Chrome浏览器驱动，配置浏览器选项 |
| `wait_for_manual_login()` | 等待用户手动完成JAccount登录 |
| `find_and_switch_to_signin_iframe()` | 查找并切换到包含签到信息的iframe |
| `get_signin_number_from_first_rows()` | 检查所有表格第一行，提取签到号 |
| `check_signin_number()` | 单次完整检查流程 |
| `run_monitoring()` | 主监控循环，定期执行检查 |
| `alert_sound()` | 发出警报声音（支持Windows/macOS/Linux） |

### 辅助模块
- `analyze_table_structure()`: 调试函数，详细分析页面表格结构
- `fallback_find_signin_number()`: 备用签到号查找方法

## ⚙️ 可调参数

在 `SigninMonitorV3.__init__()` 中可调整以下参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `check_interval` | 15 | 检查频率（秒）。**注意：** 输出示例中设置为1秒仅用于演示，实际使用建议设置为10-30秒，避免频繁请求 |
| `target_url` | `https://oc.sjtu.edu.cn/courses/...` | 目标页面地址，可根据需要修改课程ID和工具ID |

当然，你也可以在函数中 `else` 分支修改默认URL。 

如何获取监测URL？在Canvas上点击你正在上的课程，点击“点名签到”，该界面下的网址即为监测URL。

## 🚀 操作流程

### 1. 环境准备
```bash
# 安装依赖包
pip install selenium webdriver-manager
```

### 2. 运行脚本

**正常监控模式：**
```bash
python signin_monitor_v3.py
```

**调试分析模式（首次使用推荐）：**
```bash
python signin_monitor_v3.py --analyze
```

### 3. 操作步骤

1. **脚本启动**：运行脚本后，会自动打开Chrome浏览器
2. **手动登录**：
   - 浏览器访问上海交通大学Canvas页面
   - **手动完成JAccount登录**（输入用户名、密码和验证码，或jaccount二维码登录）
   - 等待页面完全加载，看到签到界面
3. **开始监控**：回到命令行窗口，按**回车键**开始监控
4. **监控运行**：脚本自动定期检查签到号变化
5. **停止监控**：按 **Ctrl+C** 停止脚本

## 🛠️ 监测逻辑解析

脚本先提取签到界面的表格所在的可能的URL，随后按照以下逻辑提取签到号：

1. **遍历所有表格**：查找页面中的所有`<table>`元素
2. **检查第一行**：对每个表格，检查第一行内容
3. **智能判断**：
   - 纯文字行（如表头）→ 跳过此表格
   - 纯数字行 → 直接作为签到号
   - 包含数字的混合行 → 提取第一个数字作为签到号
4. **备用方案**：如果所有表格第一行都是文字，使用备用查找方法

## ⚠️ 注意事项

1. **登录状态**：JAccount登录会话可能过期，长时间运行后可能需要重新登录
2. **网络要求**：确保稳定的网络连接
3. **频率设置**：检查间隔不宜过短（建议10秒以上），避免对服务器造成压力
4. **浏览器兼容**：确保Chrome浏览器已安装并更新到最新版本
5. **文件生成**：调试时会生成以下文件：
   - `iframe_content.html`：iframe页面内容（用于分析页面结构）
   - 调试模式下还会生成更详细的页面分析文件

## 🔧 故障排除

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 找不到iframe | 1. 页面未完全加载<br>2. iframe结构变化<br>3. 登录失败 | 1. 增加等待时间<br>2. 运行分析模式查看结构<br>3. 检查是否成功登录 |
| 找不到签到号 | 1. 表格结构变化<br>2. 签到号不在第一行 | 1. 查看生成的HTML文件<br>2. 调整表格解析逻辑 |
| 脚本卡住 | 1. 网络问题<br>2. 页面加载超时 | 1. 检查网络连接<br>2. 按Ctrl+C停止后重试 |
